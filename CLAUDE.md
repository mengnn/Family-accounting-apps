## 四、技术栈选型（✅ 已定稿：Electron + Vue）

跨平台（Windows + Mac）桌面应用有几条主流技术路线。下面是给小白看的对比，**最终由你选择**。详细通俗解释见对话内容与下方。

### 方案对比表

| 方案 | 上手难度 | 应用体积 | 运行流畅度 | 中文资料/求助 | 可顺便出手机版 | 我的评价 |
|---|---|---|---|---|---|---|
| ① Electron + Vue | 易 | 大（~100MB+） | 中 | 极多 | 否 | **推荐**：生态成熟、求助最容易 |
| ② Tauri + Vue | 中 | 小（~10MB） | 快 | 较多 | 否 | 现代、轻巧，需多装一个 Rust |
| ③ Flutter | 中偏难 | 中 | 快 | 一般 | 是 | 一套代码覆盖多端，但桌面端较新 |

### 各方案通俗解释

**① Electron + Vue（推荐）**
- **是什么**：用网页技术做成的桌面软件。"Vue" 是写界面的工具，"Electron" 把网页打包成可安装的 .exe / .dmg 程序。（典型代表：VS Code、飞书桌面版都是 Electron 做的）
- **优势**：① 生态最成熟，遇到任何问题都能搜到答案（中文资料极多）；② 界面好看、组件库丰富；③ 后期想改成网页版也容易。
- **劣势**：① 生成的软件体积较大（100MB 起步）；② 比较吃内存（打开后占几百 MB）。
- 适合：想稳妥、怕卡壳、想要漂亮界面的人。

**② Tauri + Vue**
- **是什么**：和方案①类似的开发方式，但用 Rust（一种现代编程语言）替代 Electron 做底层，体积和性能更好。
- **优势**：① 软件极小（约 10MB）；② 运行快、省内存；③ 更安全。
- **劣势**：① 相对较新，中文资料比 Electron 少一些；② 第一次配置需安装 Rust 工具（我代劳，但过程比 Electron 多一步）。
- 适合：在意软件小巧轻快、不介意用相对新一点技术的人。

**③ Flutter**
- **是什么**：谷歌出品的一套 UI 工具，用 Dart 语言写一次，能同时生成电脑版、手机版、网页版。
- **优势**：① 一套代码多端运行（以后想做手机版很省事）；② 界面精美、动画流畅。
- **劣势**：① 桌面端功能还在完善，偶尔有兼容小问题；② 学习资料以移动端为主，电脑端中文教程较少；③ 需学一门新语言（Dart）。
- 适合：未来想覆盖手机端、追求极致界面的人。

> 💡 **我的推荐：方案① Electron + Vue** —— 对零基础项目最稳妥，出问题最容易解决，界面也漂亮。若你很在意软件小巧，方案②也值得考虑。

### 配套技术（无论选哪个框架都一致，由我决定，无需你操心）
- **数据存储**：本地数据库 SQLite（轻量、单文件、适合记账数据；所有数据存在你电脑里，隐私安全、无需联网）
- **界面组件库**：根据所选框架搭配成熟开源组件库（如 Element Plus / Naive UI 等）
- **打包工具**：对应框架工具链，最终输出 Windows 的 .exe/.msi 与 Mac 的 .dmg 安装包

### 5.3 如何打包成可安装的安装包
- Windows 安装包：`npm run build:win` → 生成 `dist/家庭超级记账-0.1.0-x64.exe`（带安装向导）
- macOS 安装包：`npm run build:mac`（须在 Mac 电脑上执行，Windows 无法打 Mac 包）

### 5.4 已知环境坑（已固化解决，通常无需手动处理）
本机环境调试时踩过两个 Electron 经典坑，已通过脚本自动处理：

1. **`ELECTRON_RUN_AS_NODE` 环境变量**：某些工具（VSCode / Claude Code 进程）会设置它，导致 Electron 被当成纯 Node 运行，应用打不开窗口（报 `app undefined`）。
   - 解决：`scripts/dev.mjs` 启动前自动清除该变量。你用 `npm run dev` 即可，无需手动处理。

2. **better-sqlite3 原生模块 ABI 不匹配**：better-sqlite3 默认按普通 Node 编译，在 Electron 里会报 `NODE_MODULE_VERSION` 不匹配、打不开数据库。
   - 解决：`scripts/rebuild.mjs` 用 `electron-rebuild` 针对 Electron 重新编译，并通过 `postinstall` 在 `npm install` 后自动执行。
   - 若手动删 `node_modules` 后重装，`npm install` 会自动重新编译；万一报错可单独运行 `npm run rebuild`。
   - 重编译需要 Visual Studio Build Tools（C++ 桌面开发）+ Python，本机已具备。
